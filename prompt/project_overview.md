<개발하려는 아키텍쳐>

- 특정 홈페이지에서, '공적조서 작성' 버튼 클릭시 api가 실행되는 기능을 제작

- 해당 api는 '공적조서 작성'프로세스.

- 해당 api는 서버 A에서 라우팅 중

- '공적조서 작성' 클릭 시 > 로그인 창이 뜸

- 로그인 정보가 서버 A로 넘어가면, '공적조서 작성 프로세스'가 실행.

- 공적 조서 프로세스는 로그인 정보를 바탕으로 총 3개의 사이트에 로그인 후 내용을 스크래핑한 후, 해당 내용을 llm이 요약하여 문서를 생성함.

- 작성된 공적조서 내용과 3개의 사이트에서 스크래핑한 내용을 다시 프론트 or java backend쪽으로 넘겨서 UI에 띄워줌.

- 클라이언트가 작성된 default 공적조서 내용을 보고, 3개의 홈페이지의 내용 중 들어가면 좋을 내용을 각각 선택적으로 선택해서 다시 보냄

- 프로세스는 응답을 받고, 공적조서를 다시 생성

====================================================
- 이미 공적 조서 작성 프로세스는 회사 내부망에서 개발되어 있음.

- api 개설 및 프론트엔드 혹은 java backend와 통신하는 아키텍쳐 설계가 이 프로젝트의 주된 과제

- 회사 내부망에 있는 코드를 유출할 수 없어, 공적조서 작성 프로세스를 복사 붙여넣기 할 수 없음.
